# Система логирования

## Расположение логов

Все логи сохраняются в папке `logs/` в корне проекта:
- `logs/telegram-client.log.YYYY-MM-DD` - лог файлы с ротацией по дням

## Уровни логирования

- `ERROR` - критические ошибки
- `WARN` - предупреждения
- `INFO` - информационные сообщения (основные операции)
- `DEBUG` - детальная отладочная информация

## Что логируется

### Запуск приложения
- Версия приложения
- Загрузка конфигурации
- Инициализация компонентов

### Загрузка медиа (download_media)
1. Входные параметры (account_id, chat_id, message_id)
2. Получение клиента и поиск чата
3. Поиск сообщения в истории
4. Определение типа медиа и расширения файла
5. Создание директорий
6. Процесс загрузки
7. Проверка файла на диске
8. Результат операции

### Формат записи

```
2025-01-21T19:30:15.123Z INFO download_media{account_id="xxx" chat_id=123}: telegram_client::commands::media: Message 456 has media, proceeding to download
```

Каждая запись включает:
- Временную метку (UTC)
- Уровень (INFO/DEBUG/WARN/ERROR)
- Span контекст (функция и параметры)
- Модуль (путь к коду)
- Сообщение

## Как читать логи при ошибке

1. Откройте последний файл лога:
   ```bash
   tail -f logs/telegram-client.log.$(date +%Y-%m-%d)
   ```

2. Найдите строку с `===== DOWNLOAD_MEDIA CALLED =====`

3. Следите за последовательностью:
   - `Parameters:` - входные данные
   - `Getting client...` - проверка клиента
   - `Found target chat` - чат найден
   - `Found target message` - сообщение найдено
   - `Starting media download...` - начало загрузки
   - `✓ Media downloaded successfully` - успех
   - ИЛИ `✗ Failed to download media` - ошибка

4. Для детального анализа ищите строки с `ERROR` или `✗`:
   ```bash
   grep "ERROR\|✗" logs/telegram-client.log.$(date +%Y-%m-%d)
   ```

## Очистка логов

Логи автоматически ротируются по дням. Старые логи можно удалить вручную:
```bash
rm -f logs/telegram-client.log.2025-01-*
```

## Очистка медиа-кэша

Для очистки всех загруженных файлов:
```bash
rm -rf media/
```
